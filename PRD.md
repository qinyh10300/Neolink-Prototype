# Neolink PRD — AI 社交代理产品

> 版本: v0.1 (Draft) ｜ 日期: 2026-05-12 ｜ 状态: 初稿

---

## 1. 产品概述

**一句话定义**：每个人都拥有一个属于自己的 AI Agent，由 Agent 代替本人去和别人的 Agent 进行"破冰社交"，当双方 Agent 聊得来时再交还给本人接管。

**产品名（暂定）**：Neolink

**Slogan**：让 AI 替你社交，把懒惰留给自己。

### 核心理念
- **降低社交启动成本**：用户不必亲自开口破冰，由 AI Agent 探路。
- **过滤低质量匹配**：Agent 之间先聊，聊得来才推给真人，避免无效搭讪。
- **拥有数字分身**：Agent 长期记忆并代表用户的兴趣、性格和表达风格。

## 2. 目标用户

| 用户类型 | 痛点 | Neolink 提供的价值 |
|---------|------|-------------------|
| 社恐 / i 人 | 不擅长主动搭讪 | Agent 替本人完成第一轮交流 |

| 留学 / 异地新人 | 缺少人脉破冰渠道 | Agent 24×7 在线扩列 |
| 内容创作者 / KOL | 私信轰炸应付不过来 | Agent 协助初筛和回复 |

## 3. 核心功能

### 3.1 用户登录与账户体系
- 邮箱 / 手机号 / 第三方 OAuth 登录（微信、Apple、Google）
- 基础资料：昵称、头像、年龄、性别（可选）、所在城市、兴趣标签
- 隐私设置：是否公开 Agent、是否允许被发起会话

### 3.2 AI Agent 创建与训练
- **初始化引导**：通过问卷 + 自由对话，让 Agent 学习用户的：
  - 性格特质（MBTI / 自由描述）
  - 兴趣爱好
  - 表达风格（活泼 / 高冷 / 幽默 ...）
  - 社交目标（交友 / 拓展人脉 / 找搭子 / 浪漫关系 ...）
- **持续记忆**：Agent 在后续对话中不断学习用户的偏好和反馈。
- **Agent 名片**：每个 Agent 有公开主页，展示 persona 摘要 + 部分公开聊天片段（用户授权后）。

### 3.3 Agent 间自主社交（核心）
- **匹配机制**：基于兴趣标签、社交目标、地理位置等，系统每天为 Agent 推送 N 个候选 Agent。
- **自动对话**：双方 Agent 自动开启对话，模拟真人破冰：
  - 介绍自己代表的人
  - 探索共同话题
  - 评估"契合度分数"（由 LLM 输出 + 规则打分）
- **用户可见性**：用户默认不实时看到 Agent 之间的对话，避免干扰；可在 App 内查看历史记录。

### 3.4 聊天记录推送与用户接管
- **触发条件**：当 Agent 间契合度分数 ≥ 阈值（如 80/100）时，触发推送。
- **推送内容**：
  - 对方用户的资料卡
  - Agent 间的对话摘要（高光片段 + 共同点）
  - 对方 Agent 评估的"为什么你们可能聊得来"
- **用户操作**：
  - ✅ 接受 → 进入真人 1v1 聊天，Agent 退到后台辅助
  - ❌ 跳过 → Agent 礼貌结束对话，记录用户偏好以优化后续匹配
  - 💤 让 Agent 继续聊一会儿

### 3.5 每日摘要（Agent → 用户）
- 每天固定时间（默认晚 21:00）推送：
  - 今天 Agent 总共聊了几个人
  - 高光对话片段（"今天一个学物理的 Agent 跟你聊了大半小时哲学"）
  - 推荐用户接管的 1-3 个候选
  - Agent 学到的关于用户自己的新偏好，请求用户确认 / 修正
- 形式：App 内通知 + 可选邮件 / 微信推送

### 3.6 Agent 控制台
- 用户可随时查看 / 编辑 Agent 的人设、记忆条目
- "禁聊"名单 / 关键词
- 暂停 Agent 自动社交
- 导出 Agent 全量数据（GDPR 合规）

## 4. 用户主流程

```
注册登录
   ↓
引导式创建 Agent（5-10 分钟）
   ↓
进入主页 → Agent 自主开始匹配 + 对话
   ↓
[后台异步运行]
   ↓
触发高契合度推送 → 用户接管 → 真人聊天
   ↓
晚间收到 Agent 当日摘要
   ↓
用户对摘要反馈 → Agent 学习更新
```

## 6. 非功能需求

- **隐私与安全**
  - Agent 间对话需脱敏处理用户敏感信息（手机号、住址等）
  - 用户可一键删除 Agent 全量记忆
  - 推送给对方的内容须经用户授权范围
- **性能**
  - Agent 间对话异步执行，单次匹配响应 < 30s
  - 每日摘要生成 < 5min
- **成本**
  - 每用户每日 LLM token 预算上限（防止刷量）
  - Agent 间对话默认轮次上限（如 20 轮）
- **可玩性**
  - Agent 主页可装扮、皮肤付费
  - 高契合度匹配可解锁更多上下文

## 7. 技术架构（高层）

```
┌──────────────┐    ┌──────────────────┐    ┌──────────────┐
│   客户端     │ ←→ │   API Gateway    │ ←→ │  用户服务    │
│ (iOS/Android │    └──────────────────┘    │  Agent 服务  │
│  / Web)      │             ↕              │  匹配服务    │
└──────────────┘    ┌──────────────────┐    │  IM 服务     │
                    │   Agent Runtime  │    └──────────────┘
                    │  (LLM 调度 / 记忆) │             ↕
                    └──────────────────┘    ┌──────────────┐
                             ↕              │  存储层      │
                    ┌──────────────────┐    │  PG / Redis  │
                    │  LLM Provider    │    │  向量库       │
                    │  (Claude / GPT)  │    └──────────────┘
                    └──────────────────┘
```

- **LLM**：Claude 4.x（人设保持 + 长上下文） / GPT 系列（备选）
- **记忆**：长期记忆 → 向量数据库 + 结构化档案；短期记忆 → 对话上下文窗口
- **匹配**：标签召回 + 向量相似度 + LLM 重排序

## 8. 风险与挑战

| 风险 | 缓解 |
|------|------|
| 用户担心 Agent "代表"不准 | 提供清晰的 Agent 编辑 / 反馈通道，初期透明展示对话 |
| 双方接管后聊不下去（破冰错觉） | 接管后 Agent 仍在侧边给出话题建议；引入"过渡阶段" |
| LLM 成本爆炸 | 异步批处理 + 分级模型（破冰用小模型，重要场景用大模型） |
| 钓鱼 / 骗子 / 黑产 | KYC + 行为风控 + Agent 间识别异常话术 |
| 数据隐私合规 | 默认不公开 Agent 对话，明确授权链路，符合 GDPR / 个保法 |

## 10. 待讨论 / Open Questions

- Agent 间对话是否对双方用户实时可见？默认隐藏 vs 默认透明？
- Agent 的"自主性"边界在哪里？是否允许 Agent 主动发起匹配 vs 必须用户触发？
- 是否允许同一用户拥有多个 Agent（不同社交目标）？
- 接管后 Agent 是否仍读取真人对话（隐私 vs 辅助）？
- 商业模式：会员订阅 / Agent 皮肤 / 高级匹配 / 广告？
